What is Stealth NMAP?

A production-grade CLI tool built for professional penetration testing and CTF competitions

Stealth NMAP wraps the power of Nmap and its NSE scripting engine inside an intelligent Python orchestrator. Rather than running individual scans manually and stitching results together, Stealth NMAP fires up to 32 specialised scan types concurrently, parses every output file, cross-references discovered service versions against the NVD CVE database in real-time, and delivers a single prioritised exploitation brief — ready to act on.

A built-in stealth engine (T0 paranoid timing, packet fragmentation, randomised decoy IPs, DNS source-port spoofing) lets the tool operate against heavily monitored targets while maintaining full intelligence-gathering capability.

Results are visualised as an interactive BloodHound-style attack-path graph (NetworkX + Plotly), colour-coded by risk level, so attack chains are immediately obvious — even in large, complex environments.

Core Feature Set

Every component was designed and built from scratch

⚡

Multi-Threaded Scan Engine

Up to 4 parallel Nmap instances using Python's ThreadPoolExecutor. Reduces full-coverage scan time by up to 75% compared to sequential execution. Smart conflict resolution automatically disables parallelism when stealth mode is active.

🛡️

Stealth & Evasion Engine

T0 paranoid timing (5+ min between packets), packet fragmentation (-f --data-length 25), 5 randomised decoy IPs (-D RND:5), DNS source-port spoofing, and SYN/NULL/FIN/ACK/Xmas multi-vector scanning. Designed to bypass IDS/firewall detection.

🔍

Live CVE Intelligence

Queries the NVD REST API v2.0 for every discovered service version. Fetches CVSS v3.1 scores, severity ratings, and attack vectors. Falls back through multiple sources (CVEDetails, local pattern matching) if the primary API is unavailable.

🗺️

Attack-Path Graph Visualisation

Generates a BloodHound-style network graph using NetworkX and Plotly. Nodes are colour-coded by risk (red = critical, yellow = medium), edges represent exploitation paths and pivot opportunities. Exports as static PNG and interactive HTML.

💣

Searchsploit Integration

Automatically queries ExploitDB via Searchsploit for every detected service version. Extracts exploit titles, file paths, and Metasploit module names. Falls back to curated exploit-db.com/NVD search URLs when Searchsploit is unavailable.

🎯

Intelligent Recommendations Engine

Parses all scan results and generates a prioritised attack roadmap — high/medium/low priority targets, service-specific exploitation methods, and ready-to-paste tool commands (Hydra, SQLMap, WPScan, enum4linux, CrackMapExec, etc.).

🔒

Full Port Coverage

Scans all 65 535 TCP ports with --top-ports 65535 (automatically converted to -p-). Combined with --open flag to suppress noise and surface only accessible services — essential for uncovering non-standard ports.

🕸️

NSE Script Library

Orchestrates 15+ NSE scripts including ssl-heartbleed, smb-vuln-ms17-010, http-shellshock, http-sql-injection, rdp-enum-encryption, dns-brute, and full WordPress enumeration.

📊

Structured Report Generation

Automatically writes 11 categorised output files — raw scan data, CVE priority rankings, service version inventory, attack recommendations, Searchsploit results, and a full-colour HTML interactive graph — every run, with timestamped filenames.

⚙️ Concurrency Model

Uses concurrent.futures.ThreadPoolExecutor to dispatch up to 4 Nmap subprocesses simultaneously. Each scan writes its own grepable output file. The main thread collects results via as_completed() as they finish, then merges findings before the analysis phase begins.

🌐 CVE Data Pipeline

Detected service versions are passed to a CVE enrichment function that queries the NVD REST API v2.0, parses CVSS v3.1/v3.0/v2.0 scores, and falls back to CVEDetails scraping and local pattern-matching for known critical CVEs (EternalBlue, Log4Shell, BlueKeep, Zerologon, Shellshock, Heartbleed).
